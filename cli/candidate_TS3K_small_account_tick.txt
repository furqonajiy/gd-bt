# =========================================================================
# SMALL-ACCOUNT LIVE DEPLOYMENT -- TS3K  | tag TS3K   *** THE LIVE $3K BOOK ***
#
# WHAT IT IS: the SELECTED live variant of TSL18 for a $3,000 account. NOT a new
#   alpha strategy -- the EXACT TSL18 feed and geometry (C160 self-scalper:
#   RSI 70/30 + Bollinger >=0.0006 + R:R 1.2/2.5/5.0, range_to_sl / slm1.8 /
#   max-hold 150 / TP3 / trailing 0.5-0.5 after TP2), run at ONE entry per
#   signal with a daily circuit breaker and a concurrency cap, so it fits the
#   0.01-lot floor and can never lose more than a bounded slice of the day.
#
# WHY 1 ENTRY (the 2026-07-02 selection, measured on the June/$3K grid):
#   - Full 8-entry TSL18 at $3K rides a $1,235-class max DD (41% of the account)
#     and a 25-loss streak -- the 8-entry account floor is ~$4K-$8K; step up to
#     the full TSL18 book only around ~$10K (a MANUAL, capital-staged knob).
#   - 1-entry beat 2-entry (~2x June net at similar DD%): the single leg sits at
#     the shallowest ladder price (fills most often, no deeper-leg drag) and the
#     whole 1% risk concentrates in one live-sizable leg.
#   - max-open 4 beat max-open 1 decisively (strict max-open 1 suffocated the
#     feed AND read worse DD%); daily breaker 10% cut the 6-month worst-case DD
#     from -34.6% (ungated) to ~-25.6% and losing streaks from 27 to 21 at equal
#     PF. Measured June/$3K: net +$11.3k (+378%) / DD -9.7% / PF 1.64 (grid run
#     F1; pre-refresh archive -- refreshed-archive absolutes read lower, the
#     variant ORDERING is what the selection rests on).
#   See docs/SMALL_ACCOUNT_DEPLOYMENT.md for the floor math.
#
#   The small-account levers vs the TSL18 champion book (equity-RELATIVE):
#     --entries                  8 -> 1     (one live-sizable leg per signal)
#     --max-open-signals         (none) -> 4   (cap open signal GROUPS)
#     --daily-loss-limit-pct     (none) -> 0.10 (stop NEW signals after -10%/day)
#     --initial-capital          50000 -> 3000
#     (risk-budget gate not used: a 1-leg zone cannot over-risk the ladder)
#   OTHERWISE IDENTICAL to TSL18 (geometry, trailing, locks, slippage), except
#   the TP3 broker cap is intentionally removed so the runner can trail beyond
#   TP3 instead of banking the first 40 pt move and missing a 70 pt continuation.
#
# *** READ BEFORE TRUSTING THIS ***
#  1) This does NOT create a new edge. It bounds tail risk. Judge it by max
#     drawdown, worst daily loss, and SURVIVAL first -- not net profit. Validate
#     on the report (section 10) and demo.
#  2) The gates (daily-loss, max-open-signals) are enforced in BOTH the backtest
#     (run_backtest + backtest_hybrid) AND the live executor (auto, via the same
#     trading.engine DeploymentGate.live_check), proven to make identical
#     decisions (tests/test_deployment_gate.py::
#     test_live_check_matches_backtest_gate_decisions). The live daily breaker is
#     ACCOUNT-level (today's realized P&L on the symbol vs start-of-day equity),
#     which suits a dedicated small account. Still DEMO-validate first: ELEV8 ticks
#     != your broker, and the live daily-P&L read needs MT5 history.
#  3) Numbers are tick-preferred but ELEV8 ticks != your live/demo broker.
#  4) If V072 runs on the SAME account, note the gates are per-executor: TS3K's
#     max-open counts only TS3K's signals, and its ACCOUNT-level daily breaker
#     will also pause TS3K after a bad V072 day (deliberate capital protection).
#     Combined exposure runs slightly hotter than the pooled simulation, which
#     shared one cap -- set --max-open-signals 3 to be conservative.
#
# IDENTITY (tag **TS3K**, disjoint magics; never collides with TSL18/V072):
#   --strategy-tag TS3K . --positions-json positions_ts3k.json .
#   feed signals/ts3k.txt (static) + signals/ts3k_live.txt (live) .
#   reports/TS3K_2026xx (Excel) . forensic_ts3k.jsonl / notifications_ts3k.jsonl .
#   console_ts3k.txt.
#
# Alias: python cli/run.py ts3k <section>
